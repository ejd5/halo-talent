## 3. Mission 1 — Navbar Verification

### 3.1 Changes Reviewed

Two `react-hooks/set-state-in-effect` violations were fixed:

| Original | Fix | Rationale |
|----------|-----|-----------|
| `useEffect(() => setMounted(true), [])` + `if (!mounted) return null` guard | Removed entirely | `"use client"` components in App Router don't SSR — no hydration guard needed |
| `useEffect(() => { setIsOpen(false); setLangOpen(false); }, [pathname])` | Removed entirely | All Link onClick handlers already close menus; language buttons close their own dropdown |

### 3.2 Functional Verification

| Scenario | Mechanism | Verdict |
|----------|-----------|---------|
| Mobile menu opens | Toggle button → `setIsOpen(!isOpen)` | PASS |
| Mobile menu closes on link click | Every mobile `<Link>` has `onClick={() => setIsOpen(false)}` | PASS |
| Mobile menu closes on CTA click | Both mobile CTAs (`/apply`, `/login`) call `setIsOpen(false)` | PASS |
| Language dropdown opens | Desktop button toggles `langOpen` | PASS |
| Language dropdown closes on option click | Every locale button calls `setLangOpen(false)` | PASS |
| Language dropdown closes on click-outside | Overlay `<div>` at `fixed inset-0 z-10` with `onClick={() => setLangOpen(false)}` | PASS |
| Language switch works (desktop) | `window.location.href = /{code}` on button click | PASS |
| Language switch works (mobile) | Same mechanism + `setIsOpen(false)` to close mobile menu | PASS |
| Desktop nav links navigate | `<Link>` from `next/link` — client-side navigation | PASS |
| No hydration error | No `mounted` guard, no server/client mismatch possible | PASS |

### 3.3 Regression Risk: Pathname Effect Removal

The removed `useEffect(() => { setIsOpen(false); setLangOpen(false); }, [pathname])` is fully covered by event handlers:

- **Mobile menu**: Every click target in the mobile overlay calls `setIsOpen(false)` before navigation. The `<Link>` component's client-side navigation fires after onClick, so the menu closes before the route changes.
- **Language dropdown**: Desktop click-outside overlay (full viewport) intercepts any click including nav links, closing the dropdown. Mobile language buttons are separate (inside mobile overlay) and call `setIsOpen(false)`.

**Verdict: NO REGRESSION.**

## 6. Mission 4 — Privacy / Data Minimization

### 6.1 Bug Found and Fixed During QA

**Issue**: `/api/chat-ai/track` accepted arbitrary event name strings and unbounded payloads. An attacker could:
- Flood with arbitrary event names, polluting logs
- Send large payloads (no size limit)
- Inject arbitrary metadata fields

**Fix applied**: Added Zod validation with:
- **Event name allowlist**: `z.enum(Object.values(ChatAITrackingEvents))` — rejects unknown event names with 400
- **Payload field whitelist**: Only `href` (max 256 chars), `source` (max 64), `section` (max 64), `locale` (max 10), `clientTimestamp` (ISO datetime)
- **Body size limit**: Rejects requests >1KB with 413

### 6.2 Privacy Audit

| Check | Detail | Status |
|-------|--------|--------|
| PII collected | Only `href` (page path), no email, name, content, or fan data | PASS |
| Auth required | No — public landing page tracking is intentional | PASS |
| Event name validation | Zod enum against `ChatAITrackingEvents` allowlist | PASS (after fix) |
| Payload size limit | 1KB max body, per-field max lengths | PASS (after fix) |
| Payload field restriction | Only 5 safe fields allowed | PASS (after fix) |
| No Supabase write for anonymous | `trackEvent()` only persists when `userId` is provided — landing page calls omit it | PASS |
| Console log truncation | `JSON.stringify(payload).substring(0, 200)` | PASS |
| Error responses | `{ error: "..." }` only — no stack traces, no server internals | PASS |
| Response weight | `{ ok: true }` — 12 bytes | PASS |

### 6.3 Supabase Table Note

The `chat_ai_tracking_events` table referenced in `trackEvent()` requires a `user_id`. For landing page tracking (unauthenticated), no user_id is provided, so no database writes occur — events are console-logged only. This is correct for a public landing page.

**Verdict: PASS (after hardening).**
